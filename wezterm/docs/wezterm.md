## 1. WezTerm とは

[WezTerm](https://wezfurlong.org/wezterm/) は、Rust で開発された高機能ターミナルエミュレータです。  
主な特徴は以下の通りです：

- **GPU アクセラレーション**: WebGPU を利用した高速描画
- **Lua で設定**: 設定ファイルが Lua スクリプトなので、条件分岐やループなどプログラム的な制御が可能
- **タブ・ペイン・ワークスペース**: tmux なしでも画面分割やセッション管理が可能
- **ホットリロード**: 設定ファイルを保存するだけで即座に反映
- **クロスプラットフォーム**: Linux, macOS, Windows で同じ設定ファイルが使える
- **内蔵マルチプレクサ**: SSH 越しのセッション維持が可能
- **Nerd Fonts 対応**: アイコンフォントをネイティブに表示

## 2. 基本概念と用語

WezTerm を使いこなすために、以下の概念を理解しておくことが重要です。

### ウィンドウ (Window)

WezTerm のアプリケーションウィンドウそのものです。`SUPER + n` で新しいウィンドウを開けます。  
1つのウィンドウには複数の**タブ**が含まれます。

### タブ (Tab)

ブラウザのタブと同じ概念です。画面下部のタブバーに一覧が表示されます。  
各タブには1つ以上の**ペイン**が含まれます。

```
┌─────────────────────────────────────────┐
│ ウィンドウ                                │
│ ┌─────────────────────────────────────┐ │
│ │ タブ1の内容（ペインが1つ以上）          │ │
│ └─────────────────────────────────────┘ │
│ [タブ1] [タブ2] [タブ3]     ← タブバー     │
└─────────────────────────────────────────┘
```

### ペイン (Pane)

タブ内をさらに分割した領域です。1つのペインに1つのシェル（ターミナル）が動いています。  
tmux の「ペイン」と同じ概念です。

```
┌──────────┬──────────┐
│ ペイン1   │ ペイン2   │  ← 横分割
├──────────┤          │
│ ペイン3   │          │  ← 縦分割
└──────────┴──────────┘
```

### ワークスペース (Workspace)

複数のウィンドウ・タブをグループにまとめたものです。  
「プロジェクトA用のワークスペース」「メモ用のワークスペース」のように、作業を切り替えるのに使います。  
tmux の「セッション」に相当します。

### LEADER キー

tmux の prefix キーと同じ仕組みです。本設定では **`CTRL + q`** がLEADERキーです。  
LEADER を押した後、**2秒以内**に次のキーを押すことで、様々な操作を実行できます。

```
CTRL + q を押す（一度離す）→ 2秒以内に次のキーを押す
例: CTRL+q → d  =  縦分割
```

### モード

WezTerm にはいくつかの「モード」があり、モードに入ると通常のキー入力が一時的に別の動作になります。

| モード名 | 入り方 | 用途 |
|----------|--------|------|
| **通常モード** | デフォルト | 普通のターミナル操作 |
| **コピーモード** | `CTRL + SHIFT + x` | Vim のようにカーソルを動かしてテキストをコピー |
| **検索モード** | コピーモード中に `/` | テキスト検索 |
| **設定モード** | `LEADER + s` | ペインのサイズを自在にリサイズ |
| **ワークスペースモード** | `LEADER + w` | ワークスペースの選択・作成 |

### 修飾キーの表記

| 表記 | キー | 備考 |
|------|------|------|
| `SUPER` | Windows キー / Super キー | Ubuntu ではキーボード左下の旗マークキー |
| `CTRL` | Control キー | |
| `SHIFT` | Shift キー | |
| `ALT` | Alt キー | |
| `LEADER` | `CTRL + q` → 離す → 次のキー | 2秒のタイムアウトあり |

### キー表記の読み方

```
CTRL + SHIFT + x     →  3つのキーを同時に押す
LEADER + d           →  CTRL+q を押して離し、その後 d を押す
ALT + [              →  Alt と [ を同時に押す
SUPER + 1            →  Windows キーと 1 を同時に押す
```

## 3. セットアップ

### 前提条件

| ツール | 必須 | 用途 |
|--------|------|------|
| [WezTerm](https://wezfurlong.org/wezterm/) | ✅ | ターミナル本体 |
| [JetBrains Mono](https://www.jetbrains.com/mono/) | ✅ | フォント |
| [Nerd Fonts](https://www.nerdfonts.com/) | ✅ | タブやステータスバーのアイコン表示 |
| [GNU Stow](https://www.gnu.org/software/stow/) | ✅ | シンボリックリンク管理 |
| [fzf](https://github.com/junegunn/fzf) | ⚠️ 推奨 | Claude セッション切替 |
| [git](https://git-scm.com/) | ⚠️ 推奨 | ステータスバーのブランチ表示 |

### インストール手順

```bash
# 1. WezTerm のインストール (Ubuntu)
curl -fsSL https://apt.fury.io/wez/gpg.key | sudo gpg --yes --dearmor -o /usr/share/keyrings/wezterm-fury.gpg
echo 'deb [signed-by=/usr/share/keyrings/wezterm-fury.gpg] https://apt.fury.io/wez/ * *' | sudo tee /etc/apt/sources.list.d/wezterm.list
sudo apt update
sudo apt install wezterm

# 2. フォントのインストール (Ubuntu)
sudo apt install fonts-jetbrains-mono

# 3. dotfiles のリンク
cd ~/dotfiles
stow -v wezterm
```

### 設定ファイルの場所

WezTerm は以下の順序で設定ファイルを探します（先に見つかったものを使用）：

1. `~/.wezterm.lua` ← **これがあると `.config` 以下は無視されます！**
2. `~/.config/wezterm/wezterm.lua` ← 本設定はここに配置

> **注意**: ホームディレクトリ直下に `~/.wezterm.lua` がある場合は削除してください。

### 設定の反映方法

| 方法 | 操作 | 説明 |
|------|------|------|
| 自動リロード | ファイル保存 | `automatically_reload_config = true` により自動検知 |
| 手動リロード | `SUPER + r` | 強制的にリロード |
| 手動リロード | `CTRL + SHIFT + R` | 予備のリロードキー |
| 完全リセット | WezTerm を閉じて再起動 | リロードで反映されない場合に使う |

> **重要**: Lua の構文エラーがある場合、WezTerm は最後に成功した設定を使い続けます。  
> エラーがあると画面上部に赤いバナーが表示されます。

## 4. 設定ファイルの構成

```
~/.config/wezterm/
├── wezterm.lua              # エントリポイント（一般設定 + モジュール読み込み）
├── appearance.lua           # テーマ、ウィンドウ外観、タブバースタイル
├── keymaps.lua              # キーバインド定義（全モード含む）
├── statusbar.lua            # ステータスバー（Git, CPU, Mem, 時刻等）
├── tab.lua                  # タブ描画ロジック（プロセス検出, アイコン）
├── workspace.lua            # ワークスペース管理
├── theme.lua                # (レガシー: 今は appearance.lua に統合済み)
├── modules/
│   ├── opacity.lua          # フォーカス連動透過
│   ├── claude_session.lua   # Claude Code セッション切替
│   ├── aws_profile.lua      # AWS プロファイル選択
│   ├── translate.lua        # テキスト翻訳
│   ├── colorscheme.lua      # カラースキーム切替
│   ├── session.lua          # セッション保存・復元
│   └── smart_open.lua       # URL / パス / Hash の自動判別
└── scripts/
    └── preview_claude_session.lua  # fzf プレビュー用
```

### 設計パターン

すべてのモジュールは統一されたパターンで書かれています：

```lua
local module = {}

function module.apply_to_config(config)
  -- ここで config に設定を追加する
end

return module
```

`wezterm.lua` がすべてのモジュールをこの形式で読み込みます：

```lua
require("appearance").apply_to_config(config)
require("keymaps").apply_to_config(config)
-- ...
```

## 5. 基本操作

### ウィンドウ操作

| キー | 動作 |
|------|------|
| `SUPER + n` | 新しいウィンドウを開く |
| `ALT + Enter` | フルスクリーン切替 |
| `SUPER + r` | 設定ファイルをリロード |
| `CTRL + SHIFT + R` | 設定ファイルをリロード（予備） |

### デバッグ・開発ツール

| キー | 動作 |
|------|------|
| `LEADER + .` | デバッグオーバーレイ （Lua REPL + ログ閲覧）|
| `CTRL + P` | コマンドパレット（全コマンドを検索・実行） |

> **デバッグオーバーレイ**では、`wezterm.log_info()` で出力したログの確認や、  
> Lua コードのリアルタイム実行（REPL）ができます。

### コマンドパレット

`CTRL + P` で開くコマンドパレットは、WezTerm の全アクションを検索して実行できる万能ツールです。  
キーバインドを忘れた場合でも、ここからほぼ全てのコマンドにアクセスできます。

### フォントサイズ変更

画面の文字サイズを動的に変更できます。プレゼンテーション時やモニタを切り替えた時に便利です。

| キー | 動作 |
|------|------|
| `SUPER + i` | フォントを大きくする（ズームイン） |
| `SUPER + o` | フォントを小さくする（ズームアウト） |
| `SUPER + 0` | フォントサイズをリセット（デフォルトに戻す） |

> **補足**: 一般的なアプリでは `CTRL + +` / `CTRL + -` ですが、キーボード配列の違いにより  
> 日本語キーボードでは動作しないことがあるため、`SUPER + i` / `SUPER + o` を推奨します。

### その他の便利機能

| キー | 動作 |
|------|------|
| `CTRL + U` | 文字・絵文字セレクター（CharSelect）|
| `Shift + Enter` | 改行コード（`\n`）を送信 |

> **`Shift + Enter`** は Claude Code など、Enter がコマンド実行になるアプリで「改行だけしたい」時に使います。

## 6. タブ操作

タブはブラウザのタブと同じで、複数のターミナルセッションを切り替えるのに使います。  
本設定ではタブバーは**画面の下部**に表示されます。

### タブの作成と切替

| キー | 動作 |
|------|------|
| `SUPER + t` | 新しいタブを作成 |
| `SUPER + w` | 現在のタブを閉じる（確認ダイアログあり） |
| `CTRL + Tab` | 次のタブに切替 |
| `CTRL + SHIFT + Tab` | 前のタブに切替 |
| `SUPER + 1` 〜 `SUPER + 8` | タブ1〜8に直接ジャンプ |
| `SUPER + 9` | 最後のタブにジャンプ |

### タブのリネーム

| キー | 動作 |
|------|------|
| `LEADER + ,` | タブに任意のタイトルをつける |

1. `CTRL + q` を押して離す
2. `,`（カンマ）を押す
3. テキスト入力欄が表示されるので、好きな名前を入力して Enter
4. 空文字で Enter を押すと、タイトルがリセットされます

## 7. ペイン操作

ペインは、1つのタブ内を複数の領域に分割する機能です。  
tmux を使わなくても、WezTerm だけで画面分割ができます。

### ペインの作成

| キー | 動作 | イメージ |
|------|------|----------|
| `LEADER + r` | 横に分割（右に新しいペイン） | `[A] → [A][B]` |
| `LEADER + d` | 縦に分割（下に新しいペイン） | `[A] → [A]` / `[B]` |

### ペイン間の移動

| キー | 動作 |
|------|------|
| `CTRL + SHIFT + h` | 左のペインに移動 |
| `CTRL + SHIFT + j` | 下のペインに移動 |
| `CTRL + SHIFT + k` | 上のペインに移動 |
| `CTRL + SHIFT + l` | 右のペインに移動 |
| `CTRL + :` | ペイン選択UI（番号付きで選べる） |

### ペインの操作

| キー | 動作 |
|------|------|
| `LEADER + x` | 現在のペインを閉じる（確認付き） |
| `LEADER + f` | 現在のペインをズーム（全画面化 ↔ 元に戻す） |

> **ズーム**: ペインが複数ある時に `LEADER + f` を押すと、現在のペインだけが全画面になります。  
> もう一度押すと元のレイアウトに戻ります。ズーム中はタブに 🔍 アイコンが表示されます。

### ペインの最小化と復元

| キー | 動作 |
|------|------|
| `CTRL + SHIFT + c` | ペインを1行に最小化 / 元の高さに復元 |

最小化されたペインには `◀ Pane Focused ▶` というラベルが表示されます。  
もう一度同じキーを押すと、元のサイズに戻ります。

## 8. スクロールと履歴

ターミナルの出力履歴をさかのぼるための操作です。

### ページスクロール

| キー | 動作 |
|------|------|
| `SHIFT + PageUp` | 1ページ上にスクロール |
| `SHIFT + PageDown` | 1ページ下にスクロール |
| `ALT + CTRL + p` | 半ページ上にスクロール |
| `ALT + CTRL + n` | 半ページ下にスクロール |

### プロンプトジャンプ（SemanticZone）

WezTerm は Shell Integration（シェル統合）を使って、各コマンドの「プロンプト」の位置を記録しています。  
これにより、コマンドの境界を瞬時にジャンプできます。

| キー | 動作 |
|------|------|
| `ALT + [` | 前のプロンプト（コマンドの開始位置）にジャンプ |
| `ALT + ]` | 次のプロンプト（コマンドの開始位置）にジャンプ |

> **使用例**: 長い `make` の出力を見た後、`ALT + [` を何回か押すと、  
> 各コマンドの先頭に素早く戻れます。`less` や `grep` するより速い場合があります。

### マウスでのスクロール

マウスホイールでもスクロールできます。これは WezTerm のデフォルト動作です。

## 9. テキスト選択とコピー

### マウスによる選択（デフォルト動作）

| 操作 | 動作 |
|------|------|
| ドラッグ | テキストを選択 |
| ダブルクリック | 単語を選択 |
| トリプルクリック | 行全体を選択 |
| `CTRL` + クリック | URL をブラウザで開く |

### キーボードによるコピー＆ペースト

| キー | 動作 |
|------|------|
| `SUPER + c` | 選択中のテキストをクリップボードにコピー |
| `SUPER + v` | クリップボードからペースト |

### QuickSelect（後述）と コピーモード（後述）

マウスを使わずにテキストを選択する強力な方法が2つあります：
- **QuickSelect** (`SUPER + Space`): 画面内の URL やパスに素早くジャンプ
- **コピーモード** (`CTRL + SHIFT + x`): Vim のようにカーソルを動かして選択

## 10. コピーモード（Vim互換）

コピーモードは、マウスを使わずにキーボードだけでテキストを選択・コピーするための強力なモードです。  
Vim のキーバインドをベースにしています。

### コピーモードに入る・出る

| キー | 動作 |
|------|------|
| `CTRL + SHIFT + x` | コピーモードに入る |
| `q` または `Escape` | コピーモードを終了 |

コピーモード中は、画面右上に `COPY` と表示されます。

### カーソル移動

| キー | 動作 | Vim との対応 |
|------|------|-------------|
| `h` | 左に移動 | ← |
| `j` | 下に移動 | ↓ |
| `k` | 上に移動 | ↑ |
| `l` | 右に移動 | → |
| `w` | 次の単語の先頭へ | word forward |
| `b` | 前の単語の先頭へ | word backward |
| `g` | スクロールバッファの先頭（一番上）へ | `gg` 相当 |
| `G` | スクロールバッファの末尾（一番下）へ | `G` |

### ページスクロール

| キー | 動作 |
|------|------|
| `CTRL + u` | 半ページ上にスクロール |
| `CTRL + d` | 半ページ下にスクロール |

### 文字ジャンプ（f/t 系）

画面内の特定の文字めがけてカーソルを瞬時に移動させる機能です。

| キー | 動作 | 例 |
|------|------|------|
| `f` + 文字 | その文字の上に前方ジャンプ | `f(` → 次の `(` に移動 |
| `F` + 文字 | その文字の上に後方ジャンプ | `F"` → 前の `"` に移動 |
| `t` + 文字 | その文字の直前に前方ジャンプ | `t)` → 次の `)` の1つ手前に移動 |
| `T` + 文字 | その文字の直前に後方ジャンプ | |
| `;` | 直前のジャンプを同じ方向に繰り返す | |
| `,` | 直前のジャンプを逆方向に繰り返す | |

### テキスト選択

| キー | 動作 |
|------|------|
| `v` | 文字単位の選択を開始（もう一度押すと解除） |
| `V` | 行単位の選択を開始 |
| `CTRL + v` | 矩形（ブロック）選択を開始 |

1. まず `v` で選択を開始
2. `h` / `j` / `k` / `l` や `w` / `b` で範囲を広げる
3. `y` でコピー

### コピー

| キー | 動作 |
|------|------|
| `y` | 選択範囲をクリップボードにコピー（コピーモードは終了しない） |
| `Shift + Y` | 選択範囲を翻訳して表示（translate モジュール） |

### 検索との連携

| キー | 動作 |
|------|------|
| `/` | 検索モードに入る（後述） |
| `n` | 次の検索マッチへ移動 |
| `N` | 前の検索マッチへ移動 |

### 使用例：エラーメッセージをコピーする

1. `CTRL + SHIFT + x` でコピーモードに入る
2. `ALT + [` で前のコマンドのプロンプトにジャンプ（通常モード時）  
   または `g` でバッファ先頭に移動、`/error` で検索
3. `V` で行選択を開始
4. `j` を何回か押して範囲を広げる
5. `y` でコピー
6. `q` で終了

## 11. 検索モード

コピーモード中に `/` を押すと、検索モードに入ります。  
画面下部に検索バーが表示され、テキストを検索できます。

### 検索モードの操作

| キー | 動作 |
|------|------|
| 文字入力 | 検索文字列を入力 |
| `Enter` | 検索を確定してコピーモードに戻る |
| `CTRL + n` | 次のマッチに移動してコピーモードに戻る |
| `CTRL + p` | 前のマッチに移動してコピーモードに戻る |
| `Escape` | 検索をキャンセル |

### 通常モードからの検索

コピーモードに入らなくても、`SUPER + f` で直接検索を開始できます。

## 13. ワークスペース

ワークスペースは、タブやペインのセットをまるごと切り替える機能です。  
プロジェクトごとに別々の作業環境を用意し、瞬時に行き来できます。

### 特別なワークスペース

| 名前 | 用途 | 切替キー |
|------|------|----------|
| `default` | デフォルトのワークスペース | 自動 |
| `scratch` | 一時的なメモやコマンド実行用 | `CTRL + SUPER + s` |
| `nb` | ノートブック（`nb` コマンド用） | `CTRL + SUPER + a` |

> `scratch` と `nb` は「トグル」式で、押すたびに行って帰ってを繰り返します。

### ワークスペース操作

| キー | 動作 |
|------|------|
| `CTRL + SUPER + s` | scratch ワークスペースをトグル（行く/戻る） |
| `CTRL + SUPER + a` | nb ワークスペースをトグル（行く/戻る） |
| `CTRL + SUPER + n` | 次のワークスペースへ（scratch/nb をスキップ） |
| `CTRL + SUPER + p` | 前のワークスペースへ（scratch/nb をスキップ） |
| `LEADER + w` | ワークスペース一覧をfuzzy検索して選択 |

### 新しいワークスペースを作成する

1. `LEADER + w` でワークスペース選択画面を開く
2. `Shift + C` を押す
3. 名前を入力して `Enter`
4. その名前の新しいワークスペースに切り替わり、新しいタブが開きます

## 16. 拡張モジュール

### modules/session.lua — セッション保存・復元 (B-3)

`LEADER + S` (Ctrl+q + Shift+S) で現在の全ワークスペースとタブ構成を保存し、次回起動時に自動的に復元するモジュールです。

**保存される情報**:
- ワークスペースの状態
- タブの数
- 各タブのカレントディレクトリ (CWD)

**使い方**:
- **手動保存**: `LEADER + S` を押すと `~/.local/share/wezterm/sessions/last.json` に状態を書き出します。
- **自動復元**: WezTerm 起動時に最後に保存した状態を自動的にロードします。

> **注記**: 現在は「ペインの分割（スプリット）」までは保存されませんが、タブの状態とディレクトリは正確に復元されます。

### modules/opacity.lua — フォーカス連動透過

ウィンドウがフォーカスされている時と外れている時で、背景の透過度が自動的に変わります。

| 状態 | 透過度 |
|------|--------|
| フォーカス中 | 0.85 (少し透ける) |
| 非フォーカス | 0.60 (かなり透ける) |

> この機能は `wezterm.lua` 内でコメントアウト/有効化できます。

### modules/claude_session.lua — Claude Code セッション切替

`LEADER + c` で、現在実行中の Claude Code セッションを全ウィンドウ/全タブからスキャンし、  
fzf のインターフェースでプレビュー付きで切り替えることができます。

**前提**: `fzf` と `lua` がインストールされていること。

### modules/aws_profile.lua — AWS プロファイル選択

`LEADER + p` で、`~/.aws/config` に定義された AWS プロファイルを fuzzy 選択し、  
現在のシェルで `export AWS_PROFILE=選択したプロファイル` を自動実行します。

### modules/translate.lua — テキスト翻訳

コピーモード中にテキストを選択した状態で `Shift + Y` を押すと、  
選択したテキストを翻訳して別ペインに表示します。

**前提**: `plamo-translate` CLI がインストールされていること。  
インストールされていない場合は、選択テキストがそのまま表示されます。

### modules/colorscheme.lua — カラースキーム切替

`LEADER + t` で、WezTerm に内蔵されている 500 種類以上のカラースキームを fuzzy 検索し、  
**再起動なしでリアルタイムに切り替え**ることができます。

おすすめのスキーム（⭐マーク付き）がリストの先頭に表示されます。

> 切り替えたスキームは WezTerm を再起動するとデフォルト（`appearance.lua` の設定値）に戻ります。

### modules/smart_open.lua — スマート URL 開き

`LEADER + o` で、画面内の URL / ファイルパス / Git ハッシュを自動検出し、  
種類に応じて適切なアプリケーションで開きます。

config.window_background_opacity = 0.85  -- 0.0(完全透明)〜1.0(不透明)

`keymaps.lua` の以下を変更します：

local leader = { key = "q", mods = "CTRL", timeout_milliseconds = 2000 }

config.quick_select_patterns = {

  "\\b[A-Z][A-Z0-9_]+\\b",  -- 追加: 全大文字の定数名

choices = {

  { label = "Neovim" },
  { label = "Yazi" },
  { label = "Claude" },
  { label = "Htop" },      -- ← 追加
  { label = "Docker ps" }, -- ← 追加

1. `modules/` に新しいファイルを作成：

## 22. トラブルシューティング

### 設定が反映されない

1. `stow -R wezterm` でシンボリックリンクを再作成
2. `SUPER + r` でリロード
3. 画面上部に赤いバナー（エラーメッセージ）が出ていないか確認
4. それでも反映されない場合は **WezTerm を完全に閉じて再起動**

### `~/.wezterm.lua` が存在する場合

ホームディレクトリ直下に `wezterm.lua` があると、`.config/wezterm/` 以下が**完全に無視**されます。

```bash
ls -la ~/.wezterm.lua  # 存在するか確認
rm ~/.wezterm.lua       # あれば削除
```

### `SUPER + L` で画面がスリープする

これは Ubuntu の画面ロックショートカットです。WezTerm とは無関係です。  
デバッグオーバーレイを開くには、代わりに `LEADER + .` を使ってください。

### フォントズーム（`+` キー）が効かない

日本語キーボードでは `+` キーの認識が異なる場合があります。  
代わりに **`SUPER + i`**（ズームイン）/ **`SUPER + o`**（ズームアウト）を使ってください。

### カラースキーム切替が反映されない

- `appearance.lua` に `colors.background` 等のハードコードされた色指定があると、  
  カラースキームを変えても背景色が変わりません。固定色指定を削除してください。
- `modules/opacity.lua` が `set_config_overrides` を呼び出す際に、  
  既存のオーバーライド（`color_scheme` 含む）を保持するように修正済みです。
- **WezTerm の完全再起動**が必要な場合があります。

### stow でコンフリクトが発生する

```
WARNING! stowing wezterm would cause conflicts:
  * existing target is neither a link nor a directory: .config/wezterm/wezterm.lua
```

既存のファイル（シンボリックリンクではない実体ファイル）が存在する場合に発生します。  
バックアップを取ってから削除してください：

```bash
mv ~/.config/wezterm/wezterm.lua ~/.config/wezterm/wezterm.lua.bak
cd ~/dotfiles && stow -v wezterm
```

### アイコンが □（豆腐）で表示される

Nerd Fonts がインストールされていないか、フォント設定が正しくありません。

```bash
# Nerd Fonts のインストール例
wget https://github.com/ryanoasis/nerd-fonts/releases/latest/download/JetBrainsMono.zip
unzip JetBrainsMono.zip -d ~/.local/share/fonts/
fc-cache -fv
```

`wezterm.lua` のフォント設定を Nerd Font に変更：

```lua
config.font = wezterm.font("JetBrainsMono Nerd Font")
```
